## ⚡ Project Overview

<div align="center">

> ### *"Engineering intelligence into every byte — from sensor silicon to pixel-perfect UI."*

</div>

The **Humidity And Temperature Sensor System** is a **production-grade, bare-metal embedded application** engineered entirely in **Embedded C (C99)** for the **ATmega328P** 8-bit AVR microcontroller. It was designed and built from the ground up — no HAL layers, no RTOS, no abstractions — pure register-level firmware that squeezes every cycle out of the hardware.

The system acquires real-time environmental data from the **AHT20 capacitive sensor** over the **I2C/TWI bus**, processes it through a **4-state Finite State Machine (FSM)**, renders a rich **graphical interface on the 128×64 SSD1306 OLED**, controls a **variable-speed DC fan via PWM**, drives **three LED indicators**, generates **patterned buzzer alerts**, and supports **multi-screen navigation** using three physical buttons — all orchestrated by a **cooperative task scheduler** running without an RTOS.

### 📌 Why This Project Stands Out

| Aspect | Our Approach |
|--------|-------------|
| **No RTOS** | Cooperative scheduler — deterministic, lightweight, no overhead |
| **No HAL / Framework** | Pure AVR register-level programming (TWCR, TCCR1, OCR1A, DDRD…) |
| **No Floating Point** | All math via Q8.8 fixed-point — runs on 8-bit MCU with no FPU |
| **No Busy-Wait** | Timer0-driven tick, non-blocking task dispatch |
| **Full Graphical UI** | 4-screen OLED with live graphs, bar indicators, FSM badges |
| **Smart Fan Control** | Dynamic PWM duty cycle automatically mapped to temperature |
| **Robust Alerting** | 4-level FSM with hysteresis — no chattering at boundaries |

### 🌡️ AHT20 — Temperature & Humidity Sensor

| Parameter | Value |
|-----------|-------|
| Manufacturer | ASAIR (Aosong Electronics) |
| I2C Address | `0x38` (fixed) |
| Supply Voltage | 2.2V – 5.5V |
| Temperature Range | -40°C to +85°C |
| Temperature Accuracy | ±0.3°C |
| Humidity Range | 0% – 100% RH |
| Humidity Accuracy | ±2% RH |
| Raw Resolution | 20-bit output |
| Measurement Time | ~75 ms per cycle |
| Calibration | Factory calibrated (no external calibration needed) |
| Package | LCC (3 × 3 mm SMD) |

**AHT20 I2C Communication Sequence:**
```
Step 1: Power-on → Wait 40ms (sensor self-test stabilization)
Step 2: Read Status Byte (0x71)
         → If bit[3] = 0: send Init command: [0xBE, 0x08, 0x00]
         → If bit[3] = 1: sensor already calibrated, skip init
Step 3: Trigger Measurement: Write [0xAC, 0x33, 0x00] to 0x38
Step 4: Wait 80ms (ADC conversion time)
Step 5: Read 6 bytes from 0x38:
         Byte 0: Status
         Byte 1: Hum[19:12]
         Byte 2: Hum[11:4]
         Byte 3: Hum[3:0] | Temp[19:16]
         Byte 4: Temp[15:8]
         Byte 5: Temp[7:0]
Step 6: Parse:
         raw_hum  = (Byte1<<12)|(Byte2<<4)|(Byte3>>4)
         raw_temp = ((Byte3 & 0x0F)<<16)|(Byte4<<8)|Byte5
         Humidity(%)  = raw_hum  / 2^20 * 100
         Temperature  = raw_temp / 2^20 * 200 - 50
```

**SSD1306 Initialization Sequence:**
```c
// Critical commands for SSD1306 init (sent over I2C):
0xAE        // Display OFF
0xD5, 0x80  // Set Display Clock Divide / Oscillator Frequency
0xA8, 0x3F  // Set Multiplex Ratio (63 = 64 rows)
0xD3, 0x00  // Set Display Offset = 0
0x40        // Set Display Start Line = 0
0x8D, 0x14  // Charge Pump Enable
0x20, 0x00  // Memory Addressing Mode: Horizontal
0xA1        // Segment Remap (column 127 mapped to SEG0)
0xC8        // COM Output Scan Direction (remapped)
0xDA, 0x12  // COM Pins Hardware Configuration
0x81, 0xCF  // Set Contrast = 207
0xD9, 0xF1  // Pre-charge Period
0xDB, 0x40  // VCOMH Deselect Level
0xA4        // Entire Display ON (use RAM content)
0xA6        // Normal Display (not inverted)
0xAF        // Display ON
```

## 💻 Software Architecture

### 🕐 Cooperative Task Scheduler

A **lightweight cooperative (non-preemptive) task scheduler** driven by **Timer0 CTC mode** generating a precise 1 ms system tick. Each task has its own period and countdown counter. The main loop simply dispatches any task whose countdown has reached zero.

```c
// scheduler.h — Task Control Block definition
typedef struct {
    void     (*task_fn)(void);  // Pointer to task function
    uint16_t  period_ms;        // Execution period in milliseconds
    uint16_t  delay_ms;         // Time remaining until next execution
    uint8_t   run_flag;         // 1 = ready to run, 0 = waiting
    uint8_t   enabled;          // 1 = task active, 0 = suspended
} Task_t;
```

**Task Registration:**

| Task ID | Function | Period | Purpose |
|---------|----------|--------|---------|
| 0 | `Task_ReadSensor()` | 500 ms | Trigger AHT20, parse result, push to graph buffer |
| 1 | `Task_UpdateFSM()` | 500 ms | Evaluate FSM state transitions with hysteresis |
| 2 | `Task_UpdateOutputs()` | 100 ms | Sync LEDs, buzzer pattern, fan PWM to FSM state |
| 3 | `Task_RenderOLED()` | 100 ms | Clear framebuffer, draw active screen, flush to display |
| 4 | `Task_PollButtons()` | 20 ms | Debounce buttons, fire navigation events |

**Scheduler Execution Model:**
```
ISR(TIMER0_COMPA_vect)  ← Fires every 1ms (F_CPU=16MHz, OCR0A=249, prescaler=64)
    for each task:
        if task.enabled:
            task.delay_ms--
            if task.delay_ms == 0:
                task.run_flag = 1
                task.delay_ms = task.period_ms

main():
    while(1):
        SCH_Dispatch()      ← Checks run_flag for each task
            task.task_fn()  ← Execute function
            task.run_flag=0 ← Clear flag, wait for next period
```

### 🔢 Fixed-Point Q8.8 Arithmetic

ATmega328P has **no hardware FPU**. All floating-point operations would consume thousands of clock cycles via software emulation. Instead, the project uses **Q8.8 fixed-point format**: 16-bit signed integer where the lower 8 bits represent the fractional part (precision = 1/256 ≈ 0.004).

```c
// fixed_point.h — Q8.8 Fixed-Point Math Library
typedef int16_t q8_8_t;          // Q8.8 — 8 integer bits + 8 fractional bits

#define INT_TO_Q8(x)       ((q8_8_t)((x) << 8))           // 25   → 0x1900
#define Q8_TO_INT(x)       ((int16_t)((x) >> 8))          // 0x1980 → 25
#define Q8_FRAC(x)         ((uint8_t)((x) & 0xFF))        // fractional byte
#define Q8_MUL(a, b)       ((q8_8_t)(((int32_t)(a)*(b)) >> 8))
#define Q8_DIV(a, b)       ((q8_8_t)(((int32_t)(a) << 8) / (b)))

// AHT20 Temperature Conversion — pure integer, no floats:
// T(°C) = (raw_temp / 2^20) * 200 - 50
q8_8_t temp_Q8 = (q8_8_t)(((uint32_t)raw_temp * 200UL * 256UL) >> 20) - INT_TO_Q8(50);

// AHT20 Humidity Conversion:
// H(%) = (raw_hum / 2^20) * 100
q8_8_t hum_Q8 = (q8_8_t)(((uint32_t)raw_hum * 100UL * 256UL) >> 20);

// Display with 1 decimal place — integer + fractional:
// e.g., temp = 0x2080 → int=32, frac=(0x80*10/256)=5 → "32.5"
uint8_t t_int  = Q8_TO_INT(temp_Q8);
uint8_t t_frac = (uint8_t)(((uint16_t)(temp_Q8 & 0xFF) * 10) >> 8);
// → Display as "32.5°C"
```

### 🌀 Timer1 PWM Fan Control

Timer1 configured in **Phase-Correct PWM, 8-bit mode** on **OC1A (D9)**:

```c
// pwm.c
void PWM_Init(void) {
    DDRB |= (1 << PB1);                       // OC1A pin as output

    // WGM10=1 → Phase-Correct PWM, 8-bit (TOP = 0xFF)
    // COM1A1=1 → Clear OC1A on up-count match, set on down-count
    TCCR1A = (1 << COM1A1) | (1 << WGM10);

    // CS10=1 → No prescaler → f_PWM = 16MHz / (2 × 256) = 31,250 Hz
    TCCR1B = (1 << CS10);

    OCR1A = 0;                                  // Start with fan OFF
}

void PWM_SetDuty(uint8_t duty_percent) {
    // Map 0–100% to OCR1A 0–255
    OCR1A = (uint8_t)((uint16_t)duty_percent * 255U / 100U);
}
```

### 📡 I2C / TWI Register-Level Driver

```c
// i2c.c — Direct TWI hardware register access
void I2C_Init(void) {
    TWSR = 0x00;                                      // Prescaler = 1
    TWBR = ((F_CPU / 400000UL) - 16) / 2;            // 400kHz: TWBR = 12 @ 16MHz
    TWCR = (1 << TWEN);                               // Enable TWI module
}

void I2C_Start(void) {
    TWCR = (1<<TWINT)|(1<<TWSTA)|(1<<TWEN);          // Send START condition
    while (!(TWCR & (1<<TWINT)));                     // Wait for TWINT flag
    // Status: 0x08 (START) or 0x10 (REPEATED START)
}

void I2C_Stop(void) {
    TWCR = (1<<TWINT)|(1<<TWSTO)|(1<<TWEN);          // Send STOP condition
}

void I2C_Write(uint8_t data) {
    TWDR = data;                                       // Load byte into data register
    TWCR = (1<<TWINT)|(1<<TWEN);                     // Begin transmission
    while (!(TWCR & (1<<TWINT)));                     // Wait for completion
}

uint8_t I2C_ReadACK(void) {
    TWCR = (1<<TWINT)|(1<<TWEN)|(1<<TWEA);           // Read + send ACK
    while (!(TWCR & (1<<TWINT)));
    return TWDR;
}

uint8_t I2C_ReadNACK(void) {
    TWCR = (1<<TWINT)|(1<<TWEN);                     // Read + send NACK (last byte)
    while (!(TWCR & (1<<TWINT)));
    return TWDR;
}
```

### AHT20 Full Read & Parse

```c
// aht20.c — Complete measurement cycle
AHT20_Status_t AHT20_Read(q8_8_t *temp, q8_8_t *hum) {
    uint8_t buf[6];

    // 1. Trigger measurement command
    I2C_Start();
    I2C_Write(AHT20_ADDR << 1);        // 0x70 — WRITE mode
    I2C_Write(0xAC);                    // Trigger measurement
    I2C_Write(0x33);
    I2C_Write(0x00);
    I2C_Stop();

    _delay_ms(80);                      // Wait for ADC conversion

    // 2. Read 6 bytes result
    I2C_Start();
    I2C_Write((AHT20_ADDR << 1) | 1);  // 0x71 — READ mode
    buf[0] = I2C_ReadACK();             // Status byte
    buf[1] = I2C_ReadACK();             // Hum[19:12]
    buf[2] = I2C_ReadACK();             // Hum[11:4]
    buf[3] = I2C_ReadACK();             // Hum[3:0] | Temp[19:16]
    buf[4] = I2C_ReadACK();             // Temp[15:8]
    buf[5] = I2C_ReadNACK();            // Temp[7:0]
    I2C_Stop();

    if (buf[0] & 0x80) return AHT20_BUSY;  // Busy bit set

    // 3. Extract 20-bit raw values
    uint32_t raw_hum  = ((uint32_t)buf[1] << 12) |
                        ((uint32_t)buf[2] << 4)  |
                        (buf[3] >> 4);
    uint32_t raw_temp = ((uint32_t)(buf[3] & 0x0F) << 16) |
                        ((uint32_t)buf[4] << 8)  |
                        buf[5];

    // 4. Convert to Q8.8 fixed-point (no floats!)
    *hum  = (q8_8_t)(((uint32_t)raw_hum  * 100UL * 256UL) >> 20);
    *temp = (q8_8_t)(((uint32_t)raw_temp * 200UL * 256UL) >> 20) - INT_TO_Q8(50);

    return AHT20_OK;
}
```

### FSM Engine with Hysteresis

```c
// fsm.c — State machine with hysteresis-protected transitions
typedef enum { FSM_NORMAL, FSM_WARNING, FSM_CRITICAL, FSM_ALARM } fsm_state_t;
static fsm_state_t fsm_state = FSM_NORMAL;

void FSM_Update(q8_8_t temp, q8_8_t hum) {
    int16_t T = Q8_TO_INT(temp);
    int16_t H = Q8_TO_INT(hum);

    switch (fsm_state) {
        case FSM_NORMAL:
            if (T > THRESH_WARN_TEMP || H > THRESH_WARN_HUM)
                fsm_state = FSM_WARNING;
            break;

        case FSM_WARNING:
            if (T > THRESH_CRIT_TEMP || H > THRESH_CRIT_HUM)
                fsm_state = FSM_CRITICAL;
            else if (T < HYST_NORMAL_EXIT_TEMP && H < (THRESH_WARN_HUM - 2))
                fsm_state = FSM_NORMAL;
            break;

        case FSM_CRITICAL:
            if (T > THRESH_ALARM_TEMP || H > THRESH_ALARM_HUM)
                fsm_state = FSM_ALARM;
            else if (T < HYST_WARN_EXIT_TEMP && H < (THRESH_CRIT_HUM - 5))
                fsm_state = FSM_WARNING;
            break;

        case FSM_ALARM:
            if (btn_main_ack || (T < HYST_CRIT_EXIT_TEMP && H < (THRESH_ALARM_HUM - 5)))
                fsm_state = FSM_CRITICAL;
            break;
    }
    btn_main_ack = 0;
}
```

### Graph Circular Buffer + Plotter

```c
// graph.c — Ring buffer + OLED line graph renderer
#define GRAPH_BUF_SIZE  64
#define GRAPH_Y_MAX     40    // Pixel height for graph area
#define GRAPH_X_OFFSET  0
#define GRAPH_Y_OFFSET  55    // Bottom of graph area (Y increases downward)

static uint8_t  graph_buf[GRAPH_BUF_SIZE] = {0};
static uint8_t  graph_head = 0;
static uint8_t  graph_max_val = 50;   // 50°C or 100% depending on mode

void Graph_Push(uint8_t raw_val) {
    graph_buf[graph_head] = raw_val;
    graph_head = (graph_head + 1) % GRAPH_BUF_SIZE;
}

void Graph_Draw(void) {
    for (uint8_t i = 0; i < GRAPH_BUF_SIZE; i++) {
        uint8_t buf_idx = (graph_head + i) % GRAPH_BUF_SIZE;
        uint8_t val     = graph_buf[buf_idx];
        uint8_t px      = GRAPH_X_OFFSET + i;
        uint8_t py      = GRAPH_Y_OFFSET - (uint8_t)((uint16_t)val * GRAPH_Y_MAX / graph_max_val);
        SSD1306_DrawPixel(px, py, WHITE);
        SSD1306_DrawPixel(px, py + 1, WHITE);  // 2px thick line
    }
    // Draw axis lines
    SSD1306_DrawHLine(0, GRAPH_Y_OFFSET + 1, 128, WHITE);  // X axis
    SSD1306_DrawVLine(0, 15, GRAPH_Y_OFFSET, WHITE);        // Y axis
}
```
